学习汇编7

Posted on 2021-11-15

前言

先认识两个指令：G指令和P指令。

G指令：跟踪程序执行至任意一个位置。

P指令：结束程序的循环过程。

正文：

今天记录的是loop循环配合寄存器CX完成一个循环过程。首先我们知道loop的英文意思是循环的意思，所以在使用它的时候就是帮助我们执行一个循环的语句。原理如下：我们先设定CX中的值，然后在需要执行的代码前加上一个标号，具体方式为：“标号：循环需要执行的代码”。这个标号会被编译器自动编译成一个地址，然后“loop 标号”使CPU回去标号处继续执行标号后的内容，在下一次回到标号后的代码时，CX中的值会自减一，当CX中的值变为0时，循环结束。举个例子：如果我们要计算129，那么我们可以先设定这么一条语句（move CX 9），然后（s：add AX 12），接着（loop s）如此以来当程序执行完AX寄存器中的值便是129的值。当我们在调试一个loop循环时，为了避免t指令执行好几百次，我们可以使用P指令直接到达循环语句的下一个语句，也可使用G指令指向我们所要的那条代码的偏移地址。用法（G 偏移地址）以达到跟踪程序的目的。

学习汇编6

Posted on 2021-11-13

补充：在编译器中编用汇编语言编写代码，最后是以mov ax 4c00和int 21代表代码段结束。

前言：介绍在使用编译器编写代码时与debug的不同之处。

正文：在debug中，我们可以使用mov ax [0]代表把默认存在DS寄存器中的值：0这个单元内存的内容放入ax寄存器中，而在masm编译器中这条指令最终执行的结果是把0放入ax寄存器中，如果需要将内存单元为[0]的内容放入ax的话，我们需要在此条指令之前加上一条指令为：move bx 0，然后把这条指令改为move ax [bx]，此时就可以将内存单元为[0]的值放入ax中了。

注意：我们知道内存是以字为单位的，也就是两个字节，当我们要将一个数值放入内存中的时候，在寄存器中的值会从左到右先放入高位再放入地位。比如你此时在ax寄存器中的值为00be，然后把ax的值放入1000：0000的内存单元中，最后得到的结果把00放在1000：0001，把be放在1000：0000，所以在看内存的时候，注意要倒过来看。

学习汇编5

Posted on 2021-11-13

引言：本文将讲述我自己对.exe文件的执行过程的理解。

正文：如果是使用masm编译器的话，编写完一个程序，将会生成一个.asm文件，之后通过command使用masm+link或ml指令完成编译和链接成一个.exe文件。

.exe文件在执行的时候会以一个内存为256个字节的psp区域与CPU连接，让CPU执行该文件中的代码。为了区分段地址与偏移地址，假如文件开头地址为1000：0的话，那么实际上代码所在的地址为1000+10E：0。总而言之一个.exe文件的执行，必须CPU通过一段psp区域与文件产生连接，方可执行里面的代码。

第一篇博客

Posted on 2021-09-22

前言：

总结一下对于一些代码格式的理解

一：示例

首先给大家看一下以下一串代码：
#include<stdio.h>
#include<stdlib.h>
int main()
{
int a, b, c,volume,r;
printf(“请输入长宽高：”);
r = scanf_s(“%d %d %d”, &a, &b, &c);
if (r == 3 && a > 0 && b > 0 && c > 0)
volume = a * b * c;
else printf(“error\n”);
return 0;
}

然后稍微修改一下，如下：
#include<stdio.h>
#include<stdlib.h>

int main()
{
int a, b, c,volume,r;
printf(“请输入长宽高：”);
r = scanf_s(“%d %d %d”, &a, &b, &c);
if (r == 3 && a > 0 && b > 0 && c > 0)
{
volume = a * b * c;
printf(“体积是:%d\n”, volume);
}
else printf(“error\n”);
return 0;
}

二：区别

可以看到两串代码中使用(if)的时候，下面那串在if条件下有用到{}这个符号，而上面这一串代码是不需要用到的。

三：原因

是因为下面的if条件后面有用到其他的关键字，所以需要用到{}。

总结：

在用到函数的时候，如果函数里面有包含关键字，则需要用到{}，如果没有的话就可以不用。

学习汇编1

Posted on 2021-09-22 Edited on 2021-10-21

#今日学习：汇编语言
##重点内容：计算机CPU内使用CS寄存器给的索引地址*16加上IP寄存器给的地址生成一个20位的地址定位到内存的物理地址。
###前言
计算机的外部地址总线的最大寻址能力是20位，而CPU内部只允许通过16位。CPU与外部联系通过三条总线：地址总线，控制总线，数据总线。
###正文
因此，计算机CPU必须通过两段地址相加从而形成一个20位的地址通过地址总线与外部连接，才可以寻址到对应存储形式为20位的内存段。而计算机的CS与IP寄存器默认存入的16进制数为指令，因此通过CS和IP寄存器用地址总线可以连接内存当中的段并运行指令。计算机会默认把CS寄存器中的数乘上16，相当于从16位变成20位再加上IP寄存器中的值变成一个20位数索引到对应的内存地址，得以运行内存里代码。为什么寻址到相应的段的代码会被当作指令运行呢？因为CPU默认通过CS和IP寄存器寻址到的段里的代码就是要执行的指令而不是数据。
###补充
每一次寻址完把指令代码返回给CPU的预处理器以后，CPU中的IP寄存器会自动加上执行过的指令的大小从而达到可以运行下一条命令的效果
###总结
由于刚接触汇编语言，所以也不太了解，这里只是做一个我自以为的小结。计算机就是通过人为的代码输入转换成一个个对应的功能的计算机语言让CPU执行，之所以指令和数据都是一串串由0和1组成的代码，所以计算机必须通过分工来区分数据和指令。
